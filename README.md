# ライフゲージ（仮） -有限な人生を意識して、今この瞬間に行動する-

---

## サービス概要
ライフゲージは、生年月日をもとに人生の「現在地（今何%生きたか）」を算出するWebアプリです。  
残り時間をゲージ（メーター）で直感的に表示します。  
「いつかやりたい」を「今やる」に変えるきっかけを提供します。

---

## このサービスへの思い・作りたい理由
『DIE WITH ZERO』を読んだことをきっかけに、
「やりたいと思っていたことを、結局やらずに終えてしまう」
という体験を自分自身が何度もしてきたことに気づきました。

若いうちにしかできないことがあるにもかかわらず、
人は「まだ時間がある」と思い込み、行動を先延ばしにしてしまいがちです。

このサービスは、行動を管理・強制するのではなく、
人生の有限性を可視化することで、
ユーザー自身が自然と「今やろう」と思える状態を作ることを目的としています。

---

## ユーザー層について
- **20〜40代の社会人**

仕事や家庭で忙しく、やりたいことを後回しにしがちな層を想定しています。  
時間の価値を理解し始めている一方で、
行動に移す明確なきっかけを得にくいと考えたためです。

---

## サービスの利用イメージ
- 生年月日を登録すると、人生の進捗率がライフゲージとして表示される  
- やりたいことをリストとして登録し、達成したら記録できる  
- 数値と視覚表現を通して、「今この瞬間の価値」に気づく体験を提供する

- ポートフォリオとしての公開  

---

## サービスの差別化ポイント・推しポイント
- 人生をタスク管理ではなく「ゲージ」で表現する直感的なUI  
- 行動を強制せず、気づきを与える設計思想  
- 数値（進捗率）と感情（焦り・納得感）の両方に訴える表現  

### 類似サービス
- **やりたいこと管理を主目的としたサービス**（Bucket List）  
  個人のやりたいことを整理・達成するためのサービス

- **人生の進捗や残り時間を可視化するサービス**（Lifeleft / みん寿 など）  
  人生の進捗率や残り時間を数値・ビジュアルで示すことに特化したサービス

ライフゲージは、人生の進捗率という「時間の有限性」を起点に、
やりたいことを単なる理想やチェックリストとして扱うのではなく、
限られた人生の時間の中で、どのように実現していくかを具体的に考えるきっかけを提供する点に特徴があります。

## 機能候補

### MVPリリース時に作りたい機能
- ユーザー登録 / ログイン  
- 生年月日登録  
- 人生進捗率（ライフゲージ）の表示  
- やりたいことリストのCRUD  
- 達成済みフラグ・達成日の記録  

### 本リリースまでに作りたい機能
- 健康寿命を考慮したアクティブゲージの導入  
- 達成内容の共有機能  
- 人生の残り時間の減少を強調する視覚表現  

---

## 使用する技術スタック

### フレームワーク
- Ruby on Rails（7系想定）  
  - MVPで必要なCRUDや認証機能を迅速に実装できるため

### データベース
- PostgreSQL  
  - 日付や進捗データなど、将来的な拡張を見据えて採用

### デプロイ先
- Render  
  - Web Service（Railsアプリ）  
  - PostgreSQL

### 主な使用Gem

devise

tailwindcss-rails

rubocop
